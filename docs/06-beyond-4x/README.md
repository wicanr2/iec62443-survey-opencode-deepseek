# 06 · 超出核心 — 系統層、業主層與 IIoT

01–05 章把 **-4-1／-4-2 產品供應商視角**講透了。但一個組件不是活在真空裡：它被裝進**系統**、由**業主**營運、有時是**IIoT 裝置**。這一章跨出產品邊界，補上「你的組件周圍那一圈標準」——讓你知道客戶（整合商／業主）拿你的產品去滿足哪些要求，以及你的 SL-C 最終怎麼被系統的 SL-T 消費。

| 篇 | 標準 | 一句話 | 誰該讀 |
|---|---|---|---|
| [01 系統安全要求 SR](01-system-requirements-3-3.md) | -3-3:2013 | 4-2 CR 的系統層孿生兄弟：同 7 個 FR，換系統視角 | 想懂 SSA 認證、想知道系統層補償你的組件缺口 |
| [02 風險評估 ZCR](02-risk-assessment-3-2.md) | -3-2:2020 | SL-T 從哪來：把 IACS 切 zones、逐 zone 定目標等級的七步驟 | 想懂「客戶為什麼要求我 SL 3」的源頭 |
| [03 業主安全計畫](03-asset-owner-program-2-1.md) | -2-1:2024 | 業主端的安全治理：SPEs + 成熟度模型（2024 大改版） | 想懂採購方的合規框架、賣進大廠 |
| [04 IIoT 應用](04-iiot-profile-1-6.md) | -1-6:2025 (PAS) | 把 62443 套到工業物聯網：雲、閘道、大規模裝置的新考量 | 做 IIoT 裝置／閘道、追 ICSA 認證 |
